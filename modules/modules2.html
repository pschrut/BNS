MEGA BUTTON DISPLAYER

Description

The MegaButtonDisplayer is a module for making objects (buttons / links) giving a json like 'in' parameter. The module gives a div containing all buttons you want to create, rady to be inserted in the DOM.

Parameters

  • json - JSON constructed with information of the xmlOut or of a service or info tha the user knows about the element

Methods

  • createButton(button, index): Create the button/buttons with the information we pass in the json.
  • enable(idButton): Make the button enable using the id. We have to call it this way(megaButtonDisplayer instance.enable(idOfButton))
  • disable(idButton): Make the button disable using the id.We have to call it this way(megaButtonDisplayer instance.disable(idOfButton))
  • setActive(idButton): Make the button active (disabled but saying it's the active one). We have to call it this way(megaButtonDisplayer instance.disable(idOfButton))
  • updateHandler(idButton,handler){: update the Handler of the button after create it
  • updateData(idButton,data){: update the data of the button after create it
  • updateHandlerContext(idButton,handlerContext){: update the context (this) of the button after create it
  • updateEvent(idButton,event){: update the event of the button after create it
  • updateLabel(idButton,label){: update the label of the button after create it
  • getButtons(): Return the div that contains the button/buttons we want to insert in the DOM

Javascript Code

We need to create a JSON with information that will be passed to the module, it should be done using a JSON format. The parameter we can pass the JSON are:

  • elements:[]:Array that will contain all the buttons we want to display
  • defaultEventOrHandler:true if all buttons we are going to create fires an event
  • defaultButtonClassName:class we can apply all buttons we create
  • defaultDisabledClass: Class to apply by default to the disabled links
  • defaultActiveClass: Class to apply by default to the active links
  • mainClass:class of the container div of all buttons

After declare the json object, we create a veriable that contains all the information for a button. We can create a variable HTML for every button to display. These are the parameter, we can declare:

  • className: The class we want to apply to the button/link (In case we want to create a link, we have to pass 'application_action_link' plus the class we want to apply)
  • type: If is a button 'button', if is a link 'link'
  • eventOrHandler: true if the button/link fires an event
  • event: If the button/link fires an event, we pass this parameter (ej:'EWS:openApplication')
  • data: Only if the button/link fires an event, and we want to pass some arguments, we have to include this parameter
  • standardButton: In case we want to create a button (it's only for buttons) in the new way, we have to pass this button with value 'true'; If the button is a different button, containing other thing different from a label, we don't hae to pass this parameter
  • handlerContext: Context of a button/link that call a method/function, in this case is null, because we pass the context in the function
  • handler: This parameter we pass it when the button/link calls a function. handler is the name of the function (The way of the handler is this.nameOfFunction.bind(this, args); in case there are no args, we pass (this,''))
  • idButton: It will be the id of the button / link;
  • label: Text that will appearin the button /link. Another parameter to pass to the json is the type of elements to display (buttons or links)

TABLEKIT with WEB SEARCH

Description

This module allows you to create a sortable table with an input field to search in its rows

Reloading the table. Use this method: tableObject.reloadTable(domElement)

Parameters: The same as in tablekit, plus 'webSearch', 'searchLabel' and 'noResultsLabel'

  • webSearch :(boolean) true: Is used to show a filter or not
  • searchLabel :(string) 'search' label in the proper language
  • noResultsLabel :(string) 'noResults' label in the proper language
  • TABLEKIT Parameters below: You have to pass this parameters so the module will call the tablekit with them
  • marginL :(integer) Is used to position the pagination into the tFoot
  • pages : Number of lines in pages if you want pagination in tables
  • Option : Default
  • autoLoad : true: If you don't want the script to load automatically set this to false
  • stripe : true: enables row striping
  • sortable : true: enables column sorting
  • resizable : true: enables column resizing
  • editable : false: enables cell editing
  • rowEvenClass : 'table_roweven': css class used for row striping
  • rowOddClass : 'table_rowodd': css class used for row striping
  • sortableSelector : ['table.sortable']: selector for finding tables to make sortable
  • columnClass : 'table_sortcol': css class for each sortable column header cell
  • descendingClass : 'table_sortColDesc': css class for sortable column header cell when sorted descending
  • ascendingClass : 'table_sortColAsc': css class for sortable column header cell when sorted ascending
  • noSortClass : 'nosort': css class for sortable column header cell to disable sorting for that column
  • defaultSortDirection : 1: initial sort direction for all columns, 1 means ascending, -1 means desceding.
  • sortFirstAscendingClass : 'table_sortfirstasc': css class for sortable column header cell to trigger initial ascendant sorting for that column
  • sortFirstDecendingClass : 'table_sortfirstdesc': css class for sortable column header cell to trigger initial decendant sorting for that column
  • resizableSelector : ['table.resizable']: selector for finding tables to make resizable
  • minWidth : 10: minimum width in pixels of a resizable column
  • showHandle : true: shows the resize handle, with this set to false cell no handle is visible and resize is immediate with mousemove
  • resizeOnHandleClass : 'table_resize-handle-active': css class applied to header cell when the mouse pointer is in the resize handle zone (right hand cell border)
  • editableSelector : ['table.editable']: selector for finding tables to make editable
  • formClassName : 'table_editable-cell-form': css class given to the form when a cell is made editable
  • noEditClass : 'noedit': css class used on a header cell to indicate column is not editable, or on an individual body cell to indicate that cell is not editable.
  • editAjaxURI : '/': form data is POSTed to this URI from the editable cell forms
  • editAjaxOptions : {}: options for the Prototype Ajax.Updater used in editable cells

Events

  • NO events

This way the column will adapt to the text with the fixed size you have assign to the header, and you will see a text cutted, but if you put the mouse over the text, you will see the complete text

InfoPopUp

Description

The infoPopUp allows users to see some information in a ballon, and perform some actions through buttons.

Parameters

  • htmlContent: (String) The HTML content of the infoPopUp. It contains both text and buttons
  • indicatorIcon: (String) The indicator value. This can/should be one of the following values ("void", "question", "exclamation", "confirmation", "information") However, you can give a specific css class, for example, if we wanted to have the 'help' icon, we'd put indicatorIcon:'application_main_help'
  • closeButton: (Hash) The close button hash having one parameter "callBack"
  • width: (Hash) The pop up width
  • events: (Hash) The hash of events
  • showCloseButton: (Boolean) Defines if the top-right close button ('X') will be shown or not (default = true). If we put is as false, parameter 'closeButton' is not necessary

Events

  • onOpen: Fired when the confirmation box is shown on the screen
  • onClose: Fired when the confirmation box is withdrawn from the screen

BOX GRID

Description

The Box Grid module allows users to create a simple table with the posibility to have two headers (horizontal and vertical) and perform some actions on it.

Parameters

  • target: (String) Id of the element in wich the table is going to be inserted.
  • numberOfRows: (String) Number of rows for the box grid.
  • numberOfCols: (String) Number of columns for the box grid.

Methods

  • setHorTitle(target,title): Method to insert the title for the horizontal axis, out of the table.
  • setVerTitle(target,title): Method to insert the title for the vertical axis, out of the table.
  • setHeaderStyle(target,color): Method to define color of header. The user can add it in the method or insert an empty string. In this last case the color by default (grey)will be used.
  • setCellStyle(target,width,height,color): Method to define cell style(width, height and color). The three parameters are mandatory, then if the user doesn't want to change the color an empty string has to be added and the default color (white) will be used.
  • fillCell(target,html,row,col): Method to fill a cell in the row and column specified with a html structure.
  • removeCell(target,row,col): Method to remove the content of a cell in the row and column specified.
  • removeCellByContent(target,idDiv): Method to remove the content of a cell with a content specified.
  • horizontalHeader(target,headerHash): Method to insert labels in horizontal header from a hash.
  • verticalHeader(target,headerHash): Method to insert labels in vertical header from a hash.

PROGRESSBAR

Description

The progressBar module allows users to show/manage a progress bar

Parameters

  • target: (String) Id of the element in wich the progressBar is going to be inserted.
  • cellsNumber: (Number) Number of cells for the progressBar.

Important comment

We don't have a parameter to define bar's width, so it is very important to define it for its container (see example below)

Methods

  • drawSuccess(): Highlights the next empty cell as succedded (green).
  • drawFailure(): Highlights the next empty cell as failed (red).
  • reset(): Resets the bar.
  • getCellsNumber(): Returns progressBar's number of cells.
  • getCurrentCell(): Returns progressBar's current cell.

Events

  • No events

Example

Using progressBar functionality



How to

HTML Code
                <div id="progressBar1" style="width:300px;"></div>
                <input id="restart" value="Restart" type="button" onclick="javascript:reset();"/>
Javascript Code
                function start() {
                    var cells = pb.getCellsNumber();
                    var time = 5000/cells;
                    for (var i = 0; i < cells; i++) {
                        var aleat = Math.round(Math.random() * cells);
                        if (aleat%2 > 0)
                            setTimeout("pb.drawFailure()", time*(i+1));
                        else
                            setTimeout("pb.drawSuccess()", time*(i+1));
                    }
                }
                function reset() {
                    if (pb.getCellsNumber() == pb.getCurrentCell()) {
                        pb.reset();
                        start();
                    }
                }
    
                var pb = new ProgressBar({
                    target: "progressBar1",
                    cellsNumber: 15
                });
                start();
