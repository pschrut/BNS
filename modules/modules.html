MOVABLE WIDGETS

Description

Is an element of a graphical user interface (GUI) that displays an information arrangement changeable by the developer

Parameters

  • height: (integer) Used to set the height of the widget custom
  • closeButton: (boolean) On TRUE this will show the close button, on FALSE the oposite
  • minimizeButton: (boolean) On TRUE this will show the minimize button, on FALSE the oposite
  • optionsButton: (boolean) On TRUE this will show the options button, on FALSE the oposite
  • events: (Hash) The hash of events
  • semitrans: (boolean) If you set it to false, when you expand the widgets the semitransparence is not showed
  • openMinimized: (boolean) On TRUE the widget will be opened minimized (False by default)

Events

  • onClose: Fired when the widget is closed by the user
  • onMinimize: Fired when the widget is minimized
  • onMaximize: Fired when the wisget is expanded

BALLOON

Description

From now we don't need to create new balloons, because we use one balloon for the whole framework.

Parameters

  • domId: (String) The ID of the div element where the balloon will be linked to
  • content: (String) The content of the balloon
  • dimensions: (String) Dimensions of the balloon [width, height]
  • dinamicWidth: (String) Width (in pixels) for dinamic balloons - without defined dimensions
  • events: (Hash) The hash of events

Events

  • onShow: Fired when the balloon is shown
  • onHide: Fired when the balloon disappears

Example 1: Example with two buttons

The text ballon will appear near one of the following: Kate, Sawyer, Benjamin, John.

Comments on example

HOUR FIELD

Description

The hour field will draw hour / time element consisting of some input fields and will detect if the entered values are correct.

Parameters

  • viewSecs: (String) Shows the seconds input field on "yes" and doesn't on "no"
  • format: (String) Uses a 12 hour format on "12" and a 24 hour format on "24"
  • events: (Hash) The hash of events
  • defaultTime: (String) The default time that the hour input fields will contain

Events

  • onCorrectTime: Fired if the time that was entered / modified is correct
  • onIncorrectTime: Fired if the time that was entered / modified is incorrect

DATE PICKER

Description

This module creates a datepicker inside a specified div.

Parameters

  • defaultDate: (String) The date that should be pre filled in the following format: YYYYMMDD
  • fromDate: (String) The lowest date value allowed in the following format: YYYYMMDD
  • toDate: (String) The highest date value allowed in the following format: YYYYMMDD
  • manualDateInsertion: (boolean) Indicates whether the date can be modified manually. True as default.
  • draggable: (boolean) Indicates whether the date picker panel is draggable. False as default.
  • emptyDateValid: (boolean) Indicates whether the date field may be empty. True as default.
  • events: (Hash) The hash of events
  • labels: (Hash) The hash of labels ("jan", "feb", "mon", "thu", "previous", "next", ....)
  • startDay: (integer) You can choose the first day of the week
  • format: (string) If you put 'month' the format will be MM.DD.YYYY if you don't use this argument the format will be DD.MM.YYYY
  • correctDateOnBlur: (boolean) Indicates whether the correctDate event should only be fired when a field of the DatePicker loses focus, or always when the date changes. False as default.
  • fireEventOnInitialize: (boolean) Whether correctDate/wrongDate should be fired upon the initialization of the DatePicker

Methods

  • getWeek(date): Gets the week number for a given date
  • linkCalendar(calendar): Links a calendar to other (Implements the From To behaviour, It is, if you click on the first calendar on a date it will be set as minimum date on the second, and if you click on a date on the second calendar it will be set as maximum date on the second.
  • isLeapYear(year): Checks if the year is a leap year
  • updateRange(toDate,fromDate): Sets a new minimum and maximum date for the DatePicker. Parameters should come in YYYYMMDD format.
  • getDayOfWeek(day, month, year): Calculates the day of the week for a given date
  • strToDate(dateString): Converts a date string (YYYYMMDD) to a date object
  • getWeekHeader(): Gets the names of the days of the week and calculates their positions
  • loadCalendarPage(): This function fills in the calendar cells the days of a given month
  • getMonthDays(month, year): Gets the number of days for a given month in a year
  • load(): This function loads the calendar and initialize the main components
  • loadYearSelectionDialog(): This function loads the Year selection list
  • increaseYearSelectionDialog(): This function increases the year on the Year selection dialog
  • decreaseYearSelectionDialog(): This function decreases the year on the year selection dialog
  • nextMonth(): Loads next month on the calendar view
  • previousMonth(): Loads previous month on the calendar view
  • selectDay(event, day): Day clicking event (This function is triggered when the user clicks on a calendar day)
  • showYearSelection(e): This function shows the list of the years
  • showCalendar(e): This function shows and hide the calendar
  • showMonthSelection(): This functios hide or show the month selection panel depending on the current state it is
  • changeMonthByList(event, month): This function is called when the user clicks on the name of a month. The function change the calendar to the pressed month
  • changeYearByList(event, year): This function is triggered when clicking on one of the year of the year selection list
  • checkStringDateFormat(str): Replace some special characters with NaN
  • checkDateFormat(noLinkedCheck, onBlur): Checks if the date format is correct. noLinkedCheck indicates if the linked calendar should be check or no. onBlur indicates if the event that invoced checkDateFormat is from blur or no
  • getDateAsArray(): Gets the selected date as an array
  • getActualDate(): Returns the current date of the DatePicker in YYYYMMDD format.
  • initOverlay(): This function creates the overlay for the modal window that will contain the calendar
  • initCalendarDiv(): This function creates the elements needed to show the DatePicker
  • setDay(week, day, value, selected): This functions sets day number for a given week and day
  • reloadDefaultDate(): Sets the actual date to the default date, and inserts the default date on the text fields
  • checkOutside(evt): This function checks if a click is done inside 'datePicker_overlay' element or not. If the click is outside the element the function hide the calendar.
  • clearFields(): Clears day, month and year fields
  • checkDateAndFocus(e): Checks date and moves focus if necessary. e is the event that was fired to call this method. (Occurs on change of date)
  • checkDateOnBlur(e): Checks date if it was changed. e is the event that was fired to call this method. (Occurs on blur of date fields)
  • fieldWasModified(element, noLastChange): Checks if day, month or year have been modified. If so, it updates last input field. element: element to check. noLastChange: if it is true, it does not update last input.
  • moveFocusToNextField(element): The focus is set in the next element if necessary (day --> month, month --> year). element: element holding the focus
  • raiseInternalEvent(e): Raises an internal event in datePicker if it is included in the events options. e: the event raised
  • earlierThan(dP): true if the current date is earlier than the argument date, false otherwise
  • changeClassNameInAll(oldClass, newClass): Interchanges two classes for day, month and year fields
  • changeClassName(element, oldClass, newClass): Interchanges two classes for the given element
  • dateHasEmptyField(): Checks if any field is empty
  • dateIsEmpty(): Checks if all fields are empty
  • dateIsCompleted(): Checks if the date is completed, i.e., day/month/year has the minimum expected length
  • setDate(date, onBlur): Sets the current date according to the param's value. onBlur: if the date has been set as an event of blur
  • isOutOfRange(): If the date is out of range
  • showOutOfRange(): Shows the Out Of Range message under the DatePicker
  • setMonthAndDayLabels(): Sets the month and day labels
  • getLabel(): Gets the label from the options or returns the id if it does not exist

Events

  • correctDate: Fired when the selected/inserted date is correct. Returns "id", "day", "month" and "year".
  • wrongDate: Fired when the selected/inserted date is incorrect. Returns "id", "day", "month" and "year".

TABLEKIT

Description

The tableKit module allows to the users to create sortable tables

Parameters

  • marginL :(integer) Is used to position the pagination into the tFoot
  • pages : Number of lines in pages if you want pagination in tables
  • Option : Default
  • autoLoad : true: If you don't want the script to load automatically set this to false
  • stripe : true: enables row striping
  • sortable : true: enables column sorting
  • resizable : true: enables column resizing
  • editable : false: enables cell editing
  • rowEvenClass : 'table_roweven': css class used for row striping
  • rowOddClass : 'table_rowodd': css class used for row striping
  • sortableSelector : ['table.sortable']: selector for finding tables to make sortable
  • columnClass : 'table_sortcol': css class for each sortable column header cell
  • descendingClass : 'table_sortColDesc': css class for sortable column header cell when sorted descending
  • ascendingClass : 'table_sortColAsc': css class for sortable column header cell when sorted ascending
  • noSortClass : 'nosort': css class for sortable column header cell to disable sorting for that column
  • defaultSortDirection : 1: initial sort direction for all columns, 1 means ascending, -1 means desceding.
  • sortFirstAscendingClass : 'table_sortfirstasc': css class for sortable column header cell to trigger initial ascendant sorting for that column
  • sortFirstDecendingClass : 'table_sortfirstdesc': css class for sortable column header cell to trigger initial decendant sorting for that column
  • resizableSelector : ['table.resizable']: selector for finding tables to make resizable
  • minWidth : 10: minimum width in pixels of a resizable column
  • showHandle : true: shows the resize handle, with this set to false cell no handle is visible and resize is immediate with mousemove
  • resizeOnHandleClass : 'table_resize-handle-active': css class applied to header cell when the mouse pointer is in the resize handle zone (right hand cell border)
  • editableSelector : ['table.editable']: selector for finding tables to make editable
  • formClassName : 'table_editable-cell-form': css class given to the form when a cell is made editable
  • noEditClass : 'noedit': css class used on a header cell to indicate column is not editable, or on an individual body cell to indicate that cell is not editable.
  • editAjaxURI : '/': form data is POSTed to this URI from the editable cell forms
  • editAjaxOptions : {}: options for the Prototype Ajax.Updater used in editable cells

Events

  • NO events

AUTOCOMPLETER

Description

The Autocompleters Module allows to the user to choose values on an input box while the module helps to the user showing matching values retrieved from the backend

Parameters

  • label (string, null by default): a label to be shown inside the autocompleter if there is not chosen any value.
  • autoWidth (boolean, false by default): when true the text area width is auto calculated, so the whole text fits it.
  • showEverythingOnButtonClick (boolean, false by default): when true on arrow button click list shows unfiltered options list.
  • emptyOnBlur (boolean, false by default): when true the text area is cleared if it lost the focus and there's not valid text entered. When false the autocompleter will show the last valid selected option.
  • minChars (integer, 3 by default): how many characters the user has to type before the options list is shown.
  • timeout (integer, 5000 by default): number of milisecond to wait (at least) between to consecutive events to ask for new data.
  • templateOptionsList (template, '#{text} <#{data}>' by default): a string representing the template wanted to be used in options list.
  • templateResult (template, '#{text}' by default): a string representing the template wanted to be used in text area when a new result is selected.
  • splitSelectors (array, ['\\.', ',', '@'] by default): an Array of characters to be used as selectors.
  • maxShown (integer or 'auto', 'auto' by default): how many results are shown at the same time, if auto, it shows as many as div's height and scroll is enabled
  • noFilter (boolean, false by default): if true, the autocompleter won't take into account the text typed on the box, useful when we retrieve values already filtered from SAP, database...
  • events (hash, $H({}) by default): hash with the events that we want to activate
  • xmlToSend (string): Contains the XML to send to the service, with <SEARCH_PATTERN /> where the search query should go
  • searchWithService (boolean, false by default): If the search should be done through a service or no
  • url (string): URL of the service to be called
  • method (string): Method type of the service to be called (usually POST)

Events

  • onGetNewXml : Fired when the autocompleter asks for new data to the data source.
  • onResultSelected: Event fired when the user selects any value from the autocompleter.
  • onDataLoaded: Event fired when the data is loaded in the list.

Attributes inside the object that defines the autocompleter's list:

  • data
  • text
  • title (if not defined, the templateOptionsList will be used as title, it's the tooltip displayed when the mouse is over the list option)
  • def (it can be used just once, since it defines the value selected by default)

CONTEXT MENU

Description

This module provides to the user a way to display menus when a certain id is clicked.

Parameters

  • selector (string, ".contextMenu" by default): context menu will be shown when element with this id is clicked.
  • className (string, "contextMenu" by default): this is a class which will be attached to menu container (used for css styling).
  • menuItems (array, [] by default): array with items that will be shown, it can be included HTML code inside.
  • separator (boolean, false by default): when included as item on menuItems, a line will be displayed.
  • fade (boolean, false by default): when true the menu appears with a fade effect.
  • zIndex (integer, 5500 by default): with this option the developer can set the zIndex property. (see http://www.w3schools.com/Css/pr_pos_z-index.asp for further information).
  • beforeShow (function, empty function by default): function executed before the menu is shown.
  • beforeHide (function, empty function by default): function executed before the menu is hidden.
  • beforeSelect (function, empty function by default): function executed before the menu is selected.

Events

  • None

TABS

Description

This module allows to the user to display a tabs container, every tab with its own associated function

Parameters

  • labels (array, ['1','2','3','4'] by default): array with the label of every tab.
  • ids (array, [] by default): array with the id of every tab.
  • active (integer, 0 by default): index of the tab that will be active on initialization
  • callback (function, empty by default): function executed when a tab is clicked
  • mode (string ('normal' or 'max'), 'normal' by default): mode for the tabs' width, see explanation below
  • width (integer, screen/2 by default): width of the tabs container when 'max' mode is activated
  • target (string, '' by default): div's id where the tabs container will be placed.
  • firstRun (string ('y'/'n', 'n' by default): if 'yes' the callback function associated to the first active tab will be executed on initialization.
  • events (hash, $H({}) by default): hash with the events that we want to activate

Events

  • onTabClicked : Fired when the user clicks on a tab.

Comments on example

         If not defined, the 'openApplication' event is executed whith the ID field as parameter.
         The IDS options field, set the tabs ids and provides to the programmer more options.

MULTISELECT

Description

The mulstiSelect is a module for making item selections from an autocompleter. It behaves like a textarea where instead of making text insertions we insert boxes corresponding to items. Those insertions are made by using the autocompleter which will drop down a list while we are typing text. The autocompleter will be inside of the textarea-like div invisible whitout border and will be positioned after the last item box (that will simulate the textarea carret)

Parameters

  • element - The ID of the div that will behave like the textarea
  • options - An object that will set the module options, by the moment you can only set options to the autocompleter 
    {autocompleter: {showEverythingOnButtonClick: true,
                    timeout: 5000,
                    templateResult: '#{text}',
                    maxShown:5,
                    minChars: 1,
                    maxSelectedElements: 5
                }}
  • timeout: Timeout for the call
  • maxShown: Is the max Number of elements coincident, that the autocomplete can show
  • minChar: This number represents the minimum number of characters for make a call and recovery the elements.
  • maxSelectedElements: Maximum number of elements selectables in multiselect
  • JSON - The data set in JSON format. It is the data that will be searchable on the textarea, for more information about the JSON format see autocompleter module.

Methods

  • getSelect(): Returns a list of the current select items in the same format as in the data set
  • updateInput(JSON): Updates the autocompleter list with a new data set, its very usefull in the case we want to request the data at the moment of performing the search on the autocompleter, for further information about doing that see autocompleter documentation
  • disableElement(): Disables the multiselect
  • enableElement(): Enables the multiselect to add elements

Events

No events

LIST SEARCH

Description

List search is a very basic module for perform a search on a list, What it provides is a way of perform a seach on the list using a method. Once the seach is done an event is fired with the items that contains the search string.

Parameters

  • listSearch: a JSON object whith the data set to perform the seach. Format example
    •                     var data = [
                    {
                        text: 'Gustavson Johansson',
                        id: '0001'
                    },
                    {
                        text: 'Hillary Dollar',
                        id: '0002'
                    },
                    {
                        text: 'Daniel Chaplin',
                        id: '0003'
                    },
                    {
                        text: 'Pamela Jassen',
                        id: '0004'
                    }
                ];
  • options: An object with the module options
    • dataFilteredEvent: The name of the event that will be fired when performing a search. By default EWS:listSearchFiltered
    • maxNumberOfResults: The max number of results for a seach, by default -1 (All)
    • caseSensitive: The search will be case sensitive, by default false
    • returnHash: If we want to receive a Hash instead of a list as result. By default false.
    • searchField: If we want to associate a text field. It will be added the observers to control key pressing, and do the search when observing that event.

Methods

  • performSearch(searchString): This is the method for performig a search, you should pass a parameter the string to search in the list. For getting the results you should listed to the "Search Performed" defined on the options
  • getResults(): Gets the results of the last search.

Events

EWS:listSearchFiltered or user defined event - This event is fired everytime we perform a search on the list
